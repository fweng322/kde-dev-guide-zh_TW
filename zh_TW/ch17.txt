
17. 一般字彙解釋


A
Accessibility (a11y，無障礙): 讓所有人，包括身心障礙者都能使用此程式的能力。

Akonadi: KDE 上的一種可擴充，跨桌面儲存的服務，為個人資料管理（Personal Information Mangement, PIM）所設計。它提供同時讀、寫、查詢的功能。它也提供桌面物件辨識與取得的功能。

Akregator: KDE 裡一套開放源碼的 feed 閱讀器，支援 RSS 與 Atom feed 格式。Feed 可以依分類排序，並且在資料庫中可以做漸進式搜尋標題。

Algorithm (演算法): 用於計算、資料處理、自動推論的步驟程序。

Amarok: KDE 上的一套音樂管理程式，標語為：Rediscover Your Music。核心的功能包括獨一的內文瀏覽器，整合維基百科搜尋，歌詞下載，協助使用者找到新的音樂，並且找出更多樂趣。 

Application programming interface (API，應用程式介面): 一組特定的程式規則與規格，讓軟體可以遵循並彼此聯繫。它的作用就是提供在不同的軟體程式間互動的方法，類似使用者透過使用者介面與電腦程式互動的概念。

apidox: API 的文件。

Applet: 用 Java 寫成的程式，可以嵌在另一個環境，如網頁當中。

B

Backtrace (回溯追蹤): 也稱為堆疊回溯追蹤 (stack backtrace), 堆疊追蹤 (stack trace) 或是堆疊追蹤回溯 (stack traceback)。它是一份在當下時間點程式執行到哪一個點，以及目前的堆疊框架 (stack frame) 的報告。使用 KDE 軟體時，若是遇到崩潰，Dr. Konqi 會出現並提供回溯追蹤。另外，GDB 也可以用於取得回溯追蹤。

Bot: 一種軟體應用程式，跑在網際網路或 IRC 頻道中，執行一些自動化的工作。

Bug: 指程式或系統的錯誤，缺失，問題。通常程式或系統有 Bug 可能會出現不正確或未預期中的結果，或是做出非預期中的行為。

Bug Squad: 追蹤使用者回報的 KDE 軟體的問題的團隊。

Bugzilla: 一套使用網頁介面的錯誤追蹤系統與測試工具，為 KDE 社群所使用。

Build (編譯): 軟體編譯 (software build) 的簡稱，通常指將程式源碼變成系統軟體的過程或結果。軟體編譯中最重要的步驟是編譯 (compilation)，也就是將程式源碼變成可執行檔的過程。

Build system (編譯系統):
將軟體開發者每天在做的各種工作，如編譯源碼變成可執行檔，執行測試，將代碼部署到產品系統中，或是建立文件與釋出通告等，做成文稿或自動化的軟體工具。

C

Channel, IRC channel (IRC 頻道): IRC 中的聊天室，通常是一個可以詢問問題的地方。一般來說，需要尋求協助的話，不要在頻道內直接抓個人就問，這樣很不禮貌。您應該先直接發言詢問有沒有人可以幫忙。

CMake: 一套開放源碼的編譯系統。開發者只要將規格寫在文字檔中，就可以自動化編譯、測試與包裝軟體。

Code: 程式源碼。

Commit (提交): 將變更儲存進主目錄。

Compile (編譯): 用編譯器 (compiler) 處理源碼，編成執行代碼。此外也是程式或作業系統紀錄的訊息的目的地，讓管理者或開發者可以檢視這些訊息。

Console (主控台): 命令列的介面(CLI)。

C++: KDE 軟體主要使用的程式語言。

D

Debugger (除錯器): 一種軟體，用於測試與尋找其他程式裡的問題。

Desktop (桌面): 在圖形計算領域珠，桌面環境 (Desktop Environment, DE) 通常是指現代的個人電腦上所出現的圖形使用者介面所組合成的一套系統環境。目前最受歡迎的 Linux 桌面包括 KDE 工作空間與 GNOME。

Dependency (相依性): 編譯或執行您的應用程式前要先安裝的套件或軟體。

Diff:
一個用於比較檔案的工具軟體，可以輸出兩個檔案間的差異處。此外也指這種工具軟體的輸出，又稱為修補 (patch)，patch 檔可以透過 Unix 上的 patch 程式來套用差異處。

digiKam: 在 KDE 平台上開發的一套影像組織與編輯軟體。它可以在幾乎所有桌面環境與視窗管理員上執行，支援所有主要的影像格式，也可以依目錄分類相簿，或是依日期、時間、標籤等組織動態相簿。使用者也可以在影像上加上標題或評分，搜尋與儲存搜尋條件。此外還提供外掛程式，可以將相簿匯出到 23hq，Facebook，Flickr，Gallery2，Google Earth 的 KML 檔，SmugMug，Piwigo，與 Simpleviewer 等地方，或是將它們燒到光碟上，或是建立一個網頁收藏。

Distributed repository (分散式主目錄): 相對於集中式的主從架構系統，分散式主目錄是一種點對點的方式，與大家一起分享與維護程式代碼。集中式的主從架構系統中，客戶端必須與伺服器端同步。但是在分散式的架構中，每個點的工作複本都是一個獨立的主目錄，經由信任網結合在一起。

Distribution (套件，Distro): 將一群挑選過的軟體套件結合在一起，並與作業系統結合成一個系統套件。通常 GNU/Linux 特別常用這種方式。

Docs: 文件，是開發過程中的一個重要基礎。

E

EBN (English Breakfast Network): 一個用於 KDE API 文件確認、使用者文件確認、程式源碼檢查、冥想、觀星、泡茶用的站台。

Eclipse: 一套軟體整合開發環境 (IDE)，支援多種程式語言與外掛程式。它大部份是以 Java 寫成，可以用來開發 Java，不過加上不同的外掛程式之後，可以支援 Ada, C, C++, COBOL, Perl, PHP, Python, R, Ruby (包括 RoR), Scala, Clojure, Groovy, 與 Scheme 等程式語言。

Environment (環境): 設定文稿與 bash 指令的集合，提供編譯 KDE 軟體時的建議設定。

e.V. (Eingetragener Verein): 在德國的一種協會組織。KDE e.V. 是有註冊過的非營利組織，代表 KDE 社群處理法律與財務等事情。

F

Forum (論壇): 一種線上的討論站台，大家可以發文對話。這與聊天室不同的地方在於這些發出來的文可以歸檔保存。一個討論論壇通常會有類似樹狀的結構：它可以有許多子論壇，每一個子論壇有一個或數個主題。在子論壇中，每個啟始的新討論都叫一個討論串 (thread)，任何人想回覆都可以回覆。

FOSS Free and open-source software (F/OSS, FOSS) 或 free/libre/open-source software (FLOSS, FL/OSS): 自由授權的軟體。授權使用者可以透過開放出來的源碼自由使用、學習、分享、變更、改進它的設計等等。

Framework: 一種抽象的概念。實際上它是提供應用程式介面 (API) 的軟體函式庫的集合。不過在此框架的概念中，軟體提供的功能可以由使用者寫程式來改變，產生自己的軟體。

Frameworks: 從 KDE Frameworks 5.0 開始，KDE 都會有下一個主版的函式庫、執行需求的計劃，著重在模組化、釐清相依性、簡化與增進品質上。

Freenode: 一個 IRC 的網路，常用來討論各種專案。

G

GDB: GNU Debugger, GNU 軟體系統裡的標準除錯器。

Git: 一套分散式的版本控制系統，強調快速與支援在許多開發者間分享多重分支。每個 Git 工作幕錄都是個全面性的主目錄，有自己完整的歷史紀錄與版本追蹤資訊，不必依賴網路存取或與中央伺服器同步。Git 是一套以 GPL v2 散布的自由軟體。

Gluon: 一種建立與玩遊戲的方式，也是一個把遊戲的玩家與創作者集合在一起討論他們共同興趣的平台。您可以用 Gluon Creator 來建立遊戲，在 GamingFreedom.org 網站上與其他的創作者與玩家互動，並在許多支援的平台上玩 Gluon Player 的應用程式。

GPL (GNU General Public License): 一種標榜自由的反版權（copyleft，又稱「著佐權」或「公共版權」。關於 copyleft 請參考 http://www.openfoundry.org/tw/glossary/736），用於軟體與其它相關工作上。

H

I

Integrated development environment (整合開發環境，IDE): 一種軟體應用程式，提供各式各樣的功能，方便電腦程式設計師開發軟體。通常一個整合開發環境包含了源碼編輯器、編譯器或直譯器，自動編譯工具以及除錯器等元件。（也稱為整合設計環境 integrated design environment，整合除錯環境 integrated debugging environment 或互動式開發環境 interactive development environment。）

Internationalization (國際化，i18n): 一種讓程式可以容易變更不同文化與國家的介面語言的架構。請參考 Localization 的說明。

Internet Relay Chat (IRC): 一種文字介面的即時通訊工具。KDE 的頻道在：irc://irc.freenode.net。

J

K

Kate: 在 KDE 軟體集中提供的文字編輯器。Kate 是 KDE Advanced Text Editor 的頭字語。

KDE PIM (KDE Personal Information Management，KDE 個人資訊管理）: KDE 中的一個模組，包含了 Kontact, KMail, KOrganizer 等等。另外，它也表示在整個 KDE 軟體集專案中的一個工作團隊，以合作的方式負責開發個別的 Kontact 應用程式。

KDE SC (KDE Software Compilation, KDE 軟體集）：KDE 的發行套件。

KMail: KDE 的電子郵件客戶端程式，支援資料夾、信件過濾、檢視 HTML 郵件，國際字元集等等。它可以處理 IMAP, IMAP IDLE, dIMAP, POP3, 以及本地端的郵箱檔。送信的部份則支援透過 SMTP 與 sendmail 傳送信件。

Konqueror: KDE 裡的網頁瀏覽器與檔案管理員。提供檢視本地端檔案、遠端 ftp 伺服器與磁碟映像檔中的檔案的功能。

Konsole: KDE 軟體集中的一套終端模擬器。KDE 的應用程式，如 Konqueror, Krusader, Kate, Konversation, Dolphin 與 KDevelop 都將 Konsole 嵌入它們自己中。

Kontact: KDE 的個人資訊管理員與群組軟體套件。支援行事曆、聯絡人通訊錄、備忘錄、待辦事項清單、新聞與郵件功能。它也可以使用 KParts 來嵌入 Kontact 應用程式如 KMail、KAddressBook、Akregator 等）嵌入應用程式中。

Kopete: KDE 上的即時通訊應用端程式，支援多個即時通訊協定。

Kpackage Kit: PackageKit 的 KDE 前端介面。PackageKit 是一套開放源碼的套件，提供相同的前端介面，在不同系統套件裡做套件管理。

Kparts: KDE 軟體集中的組件框架。例如，Konsole 可以做為 Konqueror 與 Kate 中的一個 KPart 組件。

Konversation: 在 KDE 平台上建立的一套 IRC 客戶端程式。

KWin: KDE 軟體集中的視窗管理員。除了 KDE 外，它也可比跟其它的桌面環境搭配。

L

LAMP: 一個軟體組合的頭字語，表示 Linux, Apache, MySQL 與 Perl/PHP/Python 的組合。

Licensing (授權): 一種規範軟體使用與重新散布的法律說明。除了開放領域 (public domain) 的軟體外，所有的軟體都是透過著作權 (copyright) 保護。

Localization (地域化，l10n): 將程式介面使用的語言變成特定國家的語言與文字。請參考國際化 (Internationalization) 的說明。

M

Mailing list (郵件論壇): 將信件與資料一次傳遞給多個收件者的方式。通常由個人或組織所維護，有興趣討論的人則可以訂閱該論壇以便收到討論與資料。這個名詞也延伸到訂閱此論壇的人們。

N

Nepomuk: KDE 工作空間中支援註記、索引、搜尋與連結的專案。

Nick (暱稱): 使用者線上或螢幕上顯示的名稱。

Nightly: 每隔固定時間將版本控制系統中最新版本的源碼自動重新編譯的一種機制。這時間通常是每晚。編譯時必需在一個中性 (neutral) 的環境中，也就是說，不是用開發者所使用的環境。Neon 就是 KDE 用來做 nightly build 的專案。

O

Object-oriented programming (物件導向程式設計，OOP): 一種使用物件概念的程式設計方式。所謂的物件概念，就是在資料結構中包含了資料欄位與互動方法。

*Okular: KDE 中的文件檢視器，從 KPDF 改過來的。

Operators (IRCops): IRC 頻道的管理員，有管理頻道的權力，包括監控並將不當的使用者踢出頻道的權力。IRCops 或系統管理員可以控制 IRC 伺服器，決定誰可以加入頻道。在大部份的系統中，管理員的暱稱前面會加一個符號，不過 Freenode 通常不鼓勵管理員現身，除非他們有工作要進行。

P 

Package, packaging (套件，打包): 在 KDE 的 ftp 站上可以下載到兩種型態的套件：執行檔套件 (rpms, debs 等等) 以及源碼套件。執行檔套件是編譯過的，可執行的 KDE 軟體，可以在特定的作業系統或系統套件上執行。源碼套件就是軟體的源碼，需要編譯後才能使用。從各個系統套件庫所取得的 KDE 軟體套件會與純 KDE 源碼套件不太一樣。

Pastebin: 一個網頁應用程式，讓使用者可以上傳文字片段，如源碼的範例，供大眾檢視。通常在 IRC 頻道內要請人檢視程式時，最好使用這種方式，因為在 IRC 頻道內貼上大量文字會被認為是惡意洗板。KDE 的 pastebin 在 http://paste.kde.org

Patch (修補檔): 電腦程式的更新，或是資料更新，或是指用於修正程式用的軟體。修補通常包括安全性更新或其他的問題，以便增進可用度或效能。

Phonon: Qt 所提供的多媒體程式介面，也是 KDE 軟體集中處理多媒體串流的標準概念。

Plasma Active: Plasma 團隊最新力作，讓 KDE 的各種功能可以在行動裝置上實現。

Plasma: KDE 軟體集的大框架，提供建立各種不同元件的架構。包括互動式應用程式啟動器，視窗與工作管理員等等。

Plasmoid: Plasma 桌面環境中的元件稱為 Plasmoid。

Post-mortem debugging: 用程式崩潰後所產生的報告來除錯的動作。

Q

QtCreator: Qt SDK 裡的一個軟體，是一套為 C++ 與 JavaScript 設計的跨平台整合開發環境。

Qt: 一個跨平台的應用程式框架，用於開發有圖形使用者介面的應用軟體。

Quassel: 一套跨平台、分散式的 IRC 客戶端程式。這表示一個或多個客戶端可以附加到中央核心，或從中央核心分離。很像是用 screen 跑文字介面的客戶端程式如 WeeChat，只不過是圖形介面。

R

Rekonq: KDE 中以 WebKit 做引擎的網頁瀏覽器。

Reviewboard: 一套網頁介面的程式修補檔查驗工具，是一套以 MIT 授權發布的自由軟體。它是除了 Rietveld 與 Gerrit 的另一套方案，它可以與 Bazaar、ClearCase、CVS、Git、Mercurial、Perforce 與 Subversion 整合。

Review (查驗): 系統化地檢查程式源碼的動作。為的是在開發階段找出並修正錯誤，改進整體軟體品質並提升開發者的技能。

S

Script (文稿): 一種用文稿語言寫的小型程式，用命令直譯器 (command intepreter) 來解譯。

Server (伺服器): 用於處理其它程式 (客戶端) 所提出的請求的電腦程式。

Solid:  KDE 軟體集中的裝置整合框架。它的功能跟 KDE 的多媒體處理框架 Phonon 有點像，不過目的不是管理它自己的硬體，而是提供單一介面來存取硬體。

Source (源碼): 人類可讀的程式語言，經由編譯器、直譯器、組譯器或其它系統轉換成機器指令。

Sprint: 一群讓通常只在遠端合作開發的團隊成員見面的短期會議。

Suite: 電腦程式的集合，通常是將有相關功能的應用程式或軟體，提供一些共同的使用者介面與功能，讓彼此間可以平順地交換資料。

Summit: 讓 KDE 或其它自由軟體的開發者見面的大型會議。在此之前可能會先開小型會議 (sprint)，也可能經由 Summit 促成其它小型會議的產生。

SVN (Subversion): 一套自由授權散布的版本控制系統，是 Apache 基金會旗下的一個計畫。

T

Techbase: KDE 的開發者文件 wiki 站台。

Telepathy: 一套即時通訊系統的框架，支援許多即時通訊、VoIP 等協定。

Terminal (終端機): 序列埠的介面，顯示文字資料用。請參考主控台 (console) 的說明。

Testing: investigation conducted to provide stakeholders with information about the quality of the product or service under test.

Text editor (文字編輯器): 用於編輯純文字檔的程式。

Toolchain (工具鍊): set of programming tools that are used to create a product (typically another computer program or system of programs). The tools may be used in a chain, so that the output of each tool becomes the input for the next, but the term is used widely to refer to any set of linked development tools.

Toolkit: set of basic building units for graphical user interfaces. KDE SC uses the Qt toolkit.

Trunk: the unnamed branch (version) of a file tree under revision control. The trunk (or master) is usually meant to be the base of a project on which development progresses.

U

Unit tests: method by which individual units of source code are tested to determine if they are fit for use. A unit is the smallest testable part of an application. In object-oriented programming a unit is usually an interface, such as a class.

Usability: ease of use and learnability of a human-made object, in this case, our software.

Userbase: KDE's user documentation wiki.

V

Valgrind: GPL licensed programming tool for memory debugging, memory leak detection, and profiling. The name valgrind comes from the main entrance to Valhalla in Norse mythology.

Variable: symbolic name given to some known or unknown quantity or information, for the purpose of allowing the name to be used independently of the information it represents. A variable name in computer source code is usually associated with a data storage location and thus also its contents, and these may change during the course of program execution.

Version control: Revision control, also known as version control and source control (and an aspect of software configuration management or SCM), is the management of changes to documents, programs, and other information stored as computer files. It is most commonly used in software development, where a team of people may change the same files. Changes are usually identified by a number or letter code, termed the "revision number", "revision level", or simply "revision".

W

Widget: element of a graphical user interface (GUI) that displays an information arrangement changeable by the user, such as a window or a text box. The defining characteristic of a widget is to provide a single interaction point for the direct manipulation of a given kind of data. In other words, widgets are basic visual building blocks which, combined in an application, hold all the data processed by the application and the available interactions on this data.

Wiki: website that allows the creation and editing of any number of interlinked web pages via a web browser using a simplified markup language or a WYSIWYG text editor. Wikis are typically powered by wiki software and are often used collaboratively by multiple users. 

Word processor: computer application used for the production (including composition, editing, formatting, and possibly printing) of any sort of printable material.

X

 X, X window system: computer software system and network protocol that provides a basis for graphical user interfaces (GUIs) and rich input device capability for networked computers. It creates a hardware abstraction layer where software is written to use a generalized set of commands, allowing for device independence and reuse of programs on any computer that implements X. X.Org serves as the canonical implementation of X, and is what KDE SC uses.

Y

Yakuake: drop-down terminal emulator based on KDE Konsole technology.

Z

Z-machine: virtual machine used by Infocom for its text adventure games. Kwest is a Z-machine interpreter for KDE.
